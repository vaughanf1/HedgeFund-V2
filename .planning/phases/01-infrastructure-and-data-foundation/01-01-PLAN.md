    **backend/app/main.py**: Create FastAPI app with:
    - Lifespan context manager (placeholder for startup/shutdown)
    - GET `/health` endpoint returning `{"status": "ok", "service": "hedgefund-api"}`
    - Title: "HedgeFund V2 API"

    **backend/app/db/engine.py**: Create async engine setup following RESEARCH Pattern 2:
    - `create_async_engine` with DATABASE_URL from env
    - pool_size=20, max_overflow=0, pool_pre_ping=True, pool_recycle=3600
    - `async_sessionmaker` with `expire_on_commit=False` (CRITICAL — prevents MissingGreenlet)
    - Load DATABASE_URL from `os.environ["DATABASE_URL"]`

    **backend/app/db/models.py**: Create SQLAlchemy 2.0 ORM models using DeclarativeBase:
    1. `PriceOHLCV` — hypertable on timestamp, columns: timestamp (DateTime TZ, PK), ticker (String 20, PK), open/high/low/close (Numeric), volume (BigInteger), source (String 50)
    2. `Fundamentals` — hypertable on timestamp, columns: timestamp (PK), ticker (PK), pe_ratio, revenue, net_income, eps, debt_to_equity, free_cash_flow (all Numeric nullable), market_cap (Numeric nullable), source (String 50)
    3. `InsiderTrade` — hypertable on timestamp, columns: timestamp (PK), ticker (PK), insider_name (String 200), trade_type (String 10, "buy"/"sell"), shares (BigInteger), trade_value (Numeric), source (String 50)
    4. `NewsItem` — hypertable on timestamp, columns: timestamp (PK), ticker (PK), headline (Text), summary (Text nullable), sentiment (String 20 nullable), article_url (Text nullable), source (String 50)

    All models use `__table_args__` with `timescaledb_hypertable` annotation per RESEARCH Pattern 3. Use `Mapped[]` type annotations (SQLAlchemy 2.0 style).

    **backend/app/db/deps.py**: FastAPI dependency `get_session()` yielding AsyncSession from sessionmaker.

    **backend/alembic.ini**: Standard alembic config. Set `sqlalchemy.url` to empty string (overridden in env.py from DATABASE_URL env var). Set `script_location = alembic`.

    **backend/alembic/env.py**: CRITICAL — must include:
    1. Import `Base.metadata` from `app.db.models` as `target_metadata`
    2. The `include_object` filter from RESEARCH Pattern 4 that excludes TimescaleDB auto-created indexes
    3. Override sqlalchemy.url from `os.environ["DATABASE_URL"]` (sync version: replace `timescaledb+asyncpg` with `timescaledb+psycopg2` or just `postgresql` for Alembic which runs synchronously)
    4. Standard `run_migrations_online` and `run_migrations_offline`

    NOTE on Alembic sync driver: Alembic runs migrations synchronously. The DATABASE_URL uses asyncpg which won't work with Alembic directly. In env.py, replace the scheme: `url.replace("+asyncpg", "+psycopg2")` or use `postgresql://` prefix. Add `psycopg2-binary` to requirements.txt.

    **backend/alembic/versions/0001_initial_schema.py**: Create all four tables with hypertable conversion per RESEARCH code example. For each table:
    - `op.create_table(...)` with all columns
    - `op.execute("SELECT create_hypertable('table_name', 'timestamp', chunk_time_interval => INTERVAL '7 days')")`
    - Composite index on (ticker, timestamp) for ticker-first queries

    **backend/app/tasks/celery_app.py**: Celery app per RESEARCH code example:
    - Broker and backend from REDIS_URL env var
    - Config: task_serializer json, task_acks_late True, worker_prefetch_multiplier 1
    - beat_schedule with stubs for ingest-price (5min), ingest-fundamentals (daily 06:00 UTC), ingest-insider (daily 07:00), ingest-news (15min)
    - Task names reference `app.tasks.ingest_*.run` — these tasks will be created in Plan 01-02

    Add `psycopg2-binary` to requirements.txt for Alembic sync migrations.
