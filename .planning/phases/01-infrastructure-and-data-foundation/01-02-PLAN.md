  <action>
    **backend/app/schemas/financial.py**: Create the canonical FinancialSnapshot Pydantic v2 model from RESEARCH code example. Include:
    - `DataType = Literal["ohlcv", "fundamentals", "insider_trade", "news"]`
    - `FinancialSnapshot` with all fields (ticker, timestamp, data_type, source, plus optional fields per data type)
    - Use `Decimal` for monetary/ratio fields, `Optional` for fields that don't apply to every data type
    - Add `model_config = ConfigDict(json_encoders={Decimal: str})` (Pydantic v2 style, not inner Config class)

    **backend/app/connectors/base.py**: Create abstract DataConnector per RESEARCH Pattern 5:
    - ABC with four abstract methods: `fetch_ohlcv`, `fetch_fundamentals`, `fetch_insider_trades`, `fetch_news`
    - All methods return `list[FinancialSnapshot]` (use list, not AsyncIterator — simpler for Celery tasks which are sync)
    - Include type hints for all parameters

    **backend/app/connectors/massive.py**: Create MassiveConnector implementing DataConnector:
    - `__init__`: instantiate `RESTClient(api_key=os.environ["MASSIVE_API_KEY"])`
    - `fetch_ohlcv(ticker, from_date, to_date)`: call `self.client.list_aggs(ticker, multiplier=1, timespan="day", from_=str(from_date), to=str(to_date), limit=50000)`, map each agg to FinancialSnapshot with data_type="ohlcv", source="massive"
    - `fetch_news(ticker, limit=20)`: call `self.client.list_ticker_news(ticker=ticker, limit=limit)`, map each result to FinancialSnapshot with data_type="news", source="massive", extracting headline from `title`, summary from `description`
    - `fetch_fundamentals` and `fetch_insider_trades`: raise `NotImplementedError("Massive.com does not provide fundamentals/insider data; use FMP connector")`
    - Wrap all API calls with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=2, max=10))` from tenacity
    - NOTE: The massive client is synchronous (not async). Methods should call it synchronously.

    **backend/app/connectors/fmp.py**: Create FMPConnector implementing DataConnector:
    - `__init__`: store `api_key` from `os.environ["FMP_API_KEY"]`, create `httpx.Client()` (sync)
    - Base URL: `https://financialmodelingprep.com/api/v3`
    - `fetch_fundamentals(ticker)`: GET `/ratios-ttm/{ticker}` for PE, and `/income-statement/{ticker}?limit=1` for revenue/net_income/EPS, and `/balance-sheet-statement/{ticker}?limit=1` for debt_to_equity. Combine into single FinancialSnapshot with data_type="fundamentals", source="fmp"
    - `fetch_insider_trades(ticker, limit=50)`: GET `/insider-trading?symbol={ticker}&limit={limit}`. Map each to FinancialSnapshot with data_type="insider_trade", extracting insider_name, trade_type (map "P"/"Purchase" to "buy", "S"/"Sale" to "sell"), shares, trade_value
    - `fetch_news(ticker, limit=20)`: GET `/stock_news?tickers={ticker}&limit={limit}`. Map to FinancialSnapshot with data_type="news"
    - `fetch_ohlcv`: raise `NotImplementedError("FMP OHLCV not implemented; use Massive connector")`
    - Wrap all httpx calls with tenacity retry. Include `apikey={self.api_key}` query param on all requests.
    - Handle HTTP errors: check response.status_code, log warnings on 429 (rate limit), raise on 4xx/5xx

    IMPORTANT: FMP API confidence is LOW from research. Add clear comments marking FMP endpoint URLs as "verify against live docs" so they can be corrected during testing. Structure the code so endpoint paths are constants at the top of the file for easy modification.
  </action>

  <action>
    Create four Celery task modules, each following the same pattern:
    1. Import the Celery app from `app.tasks.celery_app`
    2. Import the relevant connector
    3. Import the relevant ORM model from `app.db.models`
    4. Import synchronous session creation (for Celery workers, which run synchronously)

    **Database session for Celery**: Celery workers are synchronous. Create a sync engine/session helper. Add to `backend/app/db/engine.py`:
    - `SYNC_DATABASE_URL` derived from DATABASE_URL by replacing `+asyncpg` with `+psycopg2` (or `timescaledb://` prefix)
    - `sync_engine = create_engine(SYNC_DATABASE_URL)`
    - `SyncSessionLocal = sessionmaker(sync_engine)`
    - This is used ONLY by Celery tasks. FastAPI uses the async engine.

    **backend/app/tasks/ingest_price.py**:
    - `@app.task(name="app.tasks.ingest_price.run")` (name must match beat_schedule)
    - Default watchlist: configurable list of tickers from env var `WATCHLIST` (comma-separated, default "AAPL,MSFT,GOOGL,AMZN,NVDA")
    - For each ticker: call `MassiveConnector().fetch_ohlcv(ticker, from_date=yesterday, to_date=today)`
    - Convert each FinancialSnapshot to PriceOHLCV model, use `session.merge()` (upsert on PK) to avoid duplicates
    - Commit batch per ticker. Log ticker, row count.
    - Wrap in try/except, log errors per ticker but continue to next ticker.

    **backend/app/tasks/ingest_fundamentals.py**:
    - `@app.task(name="app.tasks.ingest_fundamentals.run")`
    - Cache-first: before calling FMP, query DB for existing fundamentals for this ticker where timestamp > now - 24h. Skip if fresh data exists. (RESEARCH: max 1 fetch/ticker/day to avoid FMP rate limits)
    - For each ticker in watchlist: call `FMPConnector().fetch_fundamentals(ticker)`
    - Convert to Fundamentals ORM model, merge, commit.

    **backend/app/tasks/ingest_insider.py**:
    - `@app.task(name="app.tasks.ingest_insider.run")`
    - Cache-first: check DB for insider trades for this ticker where timestamp > now - 7 days. Skip if recent data exists.
    - For each ticker: call `FMPConnector().fetch_insider_trades(ticker)`
    - Convert to InsiderTrade ORM model, merge, commit.

    **backend/app/tasks/ingest_news.py**:
    - `@app.task(name="app.tasks.ingest_news.run")`
    - For each ticker: call `MassiveConnector().fetch_news(ticker)` (primary) and `FMPConnector().fetch_news(ticker)` (supplementary, in try/except)
    - Deduplicate by headline similarity before insert (simple exact-match on headline string)
    - Convert to NewsItem ORM model, merge, commit.

    ALL tasks must:
    - Use `SyncSessionLocal` (not async)
    - Have structured logging with task name, ticker, record count
    - Catch and log exceptions per-ticker without failing the whole task
    - Be importable and registered with Celery
