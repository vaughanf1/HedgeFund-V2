<objective>
Create the agent prompt infrastructure: five investor persona files with enforced information asymmetry, a DataPartitioner that builds persona-specific data contexts, a persona loader, and an LLM call wrapper with Redis-backed cost controls.

Purpose: Agent divergence is the core differentiator of this platform. If all agents see the same data, they converge to the same conclusions (sycophantic consensus). Data partitioning in the prompt architecture prevents this by design. The cost gate prevents runaway LLM spend before any agent makes a real call in Phase 3.
Output: Five versioned persona files, a data partitioner, a persona loader, and a fully cost-gated LLM wrapper.
</objective>

    **backend/app/llm/exceptions.py**: Define custom exceptions:
    - `BudgetExceededError(Exception)`: raised when daily spend limit is reached
    - `LLMCallError(Exception)`: raised on API errors from Anthropic

    **backend/app/llm/spend_tracker.py**: Create SpendTracker class per RESEARCH Pattern 7:
    - `__init__(self, redis_client, daily_limit_usd: float = None)`: Store redis client, read limit from env `DAILY_SPEND_LIMIT_USD` (default 10.0) if not provided
    - `async def get_current_spend(self) -> float`: Read today's key `llm:spend:{date.today().isoformat()}` from Redis, return float (0.0 if key missing)
    - `async def check_budget(self) -> bool`: Return True if current spend < limit
    - `async def record_spend(self, cost_usd: float) -> float`: INCRBYFLOAT on today's key, set TTL 86400, return new total
    - `async def get_daily_summary(self) -> dict`: Return `{"date": today, "spent_usd": current, "limit_usd": limit, "remaining_usd": limit - current, "budget_ok": bool}`

    Cost calculation constants (from RESEARCH):
    ```python
    COST_PER_MTOK = {
        "claude-haiku-4-5": {"input": 1.00, "output": 5.00},
        "claude-sonnet-4-6": {"input": 3.00, "output": 15.00},
    }
    ```

    **backend/app/llm/wrapper.py**: Create the gated LLM call function:
    - `async def llm_call(model: str, messages: list[dict], system: str, redis_client) -> dict`
    - Step 1: Create SpendTracker with redis_client, call check_budget(). Raise BudgetExceededError if over limit.
    - Step 2: Call `anthropic.Anthropic().messages.create(model=model, max_tokens=2048, system=system, messages=messages)`
    - Step 3: Extract usage from response: `response.usage.input_tokens`, `response.usage.output_tokens`
    - Step 4: Calculate cost using COST_PER_MTOK lookup
    - Step 5: Record spend via SpendTracker
    - Step 6: Return dict with content, model, input_tokens, output_tokens, cost_usd
    - Wrap Anthropic call in try/except for API errors, raise LLMCallError with details
    - Add structured logging: log model, token counts, cost on every call

    Also add a convenience function:
    - `async def llm_call_with_persona(persona_name: str, data_context: dict, redis_client, model: str = "claude-haiku-4-5") -> dict`
    - Loads persona via PersonaLoader, renders with data_context, calls llm_call with rendered prompt as system message and a user message like "Analyze the provided data and give your investment verdict."
    - Returns the llm_call result dict

    IMPORTANT: The Anthropic client is synchronous by default. Either:
    (a) Use `anthropic.AsyncAnthropic()` for async calls, OR
    (b) Use sync `anthropic.Anthropic()` and run in executor
    Choose (a) — use `anthropic.AsyncAnthropic()` for clean async integration with FastAPI.

    IMPORTANT: Make the wrapper the ONLY way to call the LLM. Add a module docstring: "ALL LLM calls MUST go through this wrapper. Direct use of anthropic SDK is prohibited — it bypasses cost tracking."
  </action>
