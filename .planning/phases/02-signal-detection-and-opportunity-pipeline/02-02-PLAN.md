**insider_cluster.py** (SGNL-03):
- `from __future__ import annotations`
- Read `MIN_INSIDER_CLUSTER_SIZE` from env (default "2")
- Read `INSIDER_CLUSTER_WINDOW_DAYS` from env (default "30")
- `INSIDER_CLUSTER_SQL` as `text(...)`: SELECT ticker, COUNT(DISTINCT insider_name) AS unique_buyers, SUM(CASE WHEN trade_type = 'buy' THEN shares ELSE 0 END) AS total_shares_bought, MIN(timestamp) AS first_buy, MAX(timestamp) AS last_buy FROM insider_trades WHERE ticker = :ticker AND trade_type = 'buy' AND timestamp >= NOW() - INTERVAL ':window_days days' ... HAVING COUNT(DISTINCT insider_name) >= :min_insiders.
- IMPORTANT: The INTERVAL cannot use a parameter directly. Build the SQL string dynamically with the window days injected as a Python f-string into the text() call, NOT as a bind parameter. The min_insiders IS a bind parameter.
- `detect_insider_cluster(session, ticker: str) -> dict | None`: execute SQL, if no row return None. Score = min(unique_buyers / 4.0, 1.0) — normalized so 4+ insiders = max score. Return signal dict with detail: unique_buyers, total_shares_bought, first_buy, last_buy.

**news_catalyst.py** (SGNL-04):
- `from __future__ import annotations`
- Read `NEWS_CATALYST_KEYWORDS` from env (default "earnings,surprise,partnership,acquisition,FDA,approval,revenue,upgrade")
- Parse keywords by splitting on comma, stripping whitespace.
- Build ILIKE clause dynamically: for each keyword, generate `headline ILIKE '%keyword%'`, join with OR.
- `NEWS_CATALYST_SQL`: construct with `text(...)` where the ILIKE clause is injected as a Python string (NOT a bind param — ILIKE patterns cannot be parameterized as arrays in PostgreSQL). Query: SELECT ticker, COUNT(*) AS recent_articles, MAX(timestamp) AS most_recent, EXTRACT(EPOCH FROM (NOW() - MAX(timestamp))) / 3600 AS hours_since_latest FROM news_items WHERE ticker = :ticker AND timestamp >= NOW() - INTERVAL '48 hours' AND ({ilike_clause}) GROUP BY ticker HAVING COUNT(*) >= 1.
- CRITICAL: Do NOT filter on `sentiment` column. It is unpopulated (research open question #1). Keyword-only filtering.
- `detect_news_catalyst(session, ticker: str) -> dict | None`: execute SQL, if no row return None. Score = min(recent_articles / 3.0, 1.0) with recency decay: multiply by max(0.2, 1.0 - hours_since_latest / 48.0). Return signal dict with detail: recent_articles, hours_since_latest, keywords_used.

**scorer.py** (SGNL-06):
- `from __future__ import annotations`
- Read weights from env vars: WEIGHT_VOLUME_SPIKE (0.25), WEIGHT_PRICE_BREAKOUT (0.25), WEIGHT_INSIDER_CLUSTER (0.20), WEIGHT_NEWS_CATALYST (0.20), WEIGHT_SECTOR_MOMENTUM (0.10).
- Store in `WEIGHTS: dict[str, float]` keyed by signal_type string.
- `compute_composite_score(signals: list[dict]) -> float`:
  - If `not signals`: return 0.0 (guard against empty list — Pitfall 4).
  - `total_weight = sum(WEIGHTS[s["signal_type"]] for s in signals if s["signal_type"] in WEIGHTS)`
  - If `total_weight == 0`: return 0.0
  - `weighted_sum = sum(s["score"] * WEIGHTS.get(s["signal_type"], 0.0) for s in signals)`
  - Return `weighted_sum / total_weight`

**quality_gate.py** (SGNL-07):
- `from __future__ import annotations`
- Read `SIGNAL_QUALITY_GATE` from env (default "0.35").
- `passes_gate(composite_score: float) -> bool`: return `composite_score >= QUALITY_GATE_THRESHOLD`.
