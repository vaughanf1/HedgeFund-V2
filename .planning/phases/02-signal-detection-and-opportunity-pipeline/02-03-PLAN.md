**queue.py:**
- `from __future__ import annotations`
- Import json, os, redis.
- Read `REDIS_URL` from env (default "redis://redis:6379/0").
- Read `OPPORTUNITY_DEDUP_TTL_SECONDS` from env (default "3600" — 1 hour).
- Define `OPP_QUEUE_KEY = "opportunity_queue"`.
- `enqueue_opportunity(r: redis.Redis, ticker: str, payload: dict) -> bool`:
  1. `dedup_key = f"opp:dedup:{ticker}"`
  2. `is_new = r.set(dedup_key, "1", nx=True, ex=DEDUP_TTL_SECONDS)` — atomic SET NX with TTL.
  3. If `not is_new`: return False (duplicate within window).
  4. `r.rpush(OPP_QUEUE_KEY, json.dumps(payload))` — push to Redis list.
  5. Return True.

Intent comment at top: "Phase 3 agents consume from OPP_QUEUE_KEY via BLPOP. Dedup key is per-ticker (not per-signal-type) because agents analyze the full opportunity for a ticker."

**Update scan_market.py:**
- Add import: `from app.signals.queue import enqueue_opportunity`.
- In the per-ticker loop, after the gate check: if `gate_passed` is True, call `enqueued = enqueue_opportunity(r, ticker, {"ticker": ticker, "composite_score": composite, "signals": raw_signals, "detected_at": detected_at.isoformat()})`.
- If `enqueued` is True, increment `passed` counter and log info.
- If `enqueued` is False (duplicate), log info "duplicate — skipped" but do NOT increment passed.
- Keep the existing `rejected` counter for tickers that fail the gate.

Note: The `raw_signals` list contains dicts with `detail` sub-dicts. When JSON-serializing for the queue, ensure `detail` values are JSON-serializable (they should already be — they contain floats and strings). If `Decimal` objects appear, cast them to `float` before serializing. Add a helper or use `json.dumps(payload, default=str)` as a safety net.

**signals.py** (VIS-01, VIS-04):
- `from __future__ import annotations`
- Import: APIRouter, Depends, Query from fastapi; AsyncSession from sqlalchemy.ext.asyncio; select, desc from sqlalchemy; get_session from app.db.deps; DetectedSignal from app.db.models; BaseModel from pydantic; Decimal from decimal; datetime from datetime; Optional from typing.

- Define `SignalResponse(BaseModel)`:
  - detected_at: datetime
  - ticker: str
  - signal_type: str
  - score: Decimal
  - composite_score: Optional[Decimal]
  - passed_gate: bool
  - detail: Optional[str]
  - `model_config = {"from_attributes": True}` (Pydantic v2 style, NOT class Config).

- `router = APIRouter(prefix="/api/v1/signals", tags=["signals"])`

- `@router.get("", response_model=list[SignalResponse])`:
  `async def list_signals(ticker: Optional[str] = Query(None), signal_type: Optional[str] = Query(None), passed_gate: Optional[bool] = Query(None), limit: int = Query(50, le=500), session: AsyncSession = Depends(get_session)):`
  - Build query: `select(DetectedSignal).order_by(desc(DetectedSignal.detected_at)).limit(limit)`
  - If ticker: add `.where(DetectedSignal.ticker == ticker)`
  - If signal_type: add `.where(DetectedSignal.signal_type == signal_type)`
  - If passed_gate is not None: add `.where(DetectedSignal.passed_gate == passed_gate)`
  - Execute and return `result.scalars().all()`

- `@router.get("/{ticker}", response_model=list[SignalResponse])`:
  `async def signals_for_ticker(ticker: str, limit: int = Query(20, le=200), session: AsyncSession = Depends(get_session)):`
  - Query DetectedSignal WHERE ticker == ticker, ORDER BY detected_at DESC, LIMIT limit.
  - Execute and return scalars.
