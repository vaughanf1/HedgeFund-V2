  <action>
    Create the frontend directory at project root (sibling to backend/).

    1. Scaffold with Vite: `npm create vite@latest frontend -- --template react-ts` (or manually create the files if the CLI is problematic in non-interactive mode). The key files needed:
       - `package.json` with react, react-dom, typescript, vite, @vitejs/plugin-react
       - `tsconfig.json` and `tsconfig.app.json` with strict mode, path alias `@/` -> `src/`
       - `vite.config.ts` with react plugin and path alias resolve
       - `index.html` with root div
       - `src/main.tsx` rendering App into root

    2. Install core dependencies:
       ```
       npm install @xyflow/react zustand @dagrejs/dagre
       npm install -D tailwindcss @tailwindcss/vite
       ```

    3. Configure Tailwind v4:
       - Add `@tailwindcss/vite` plugin to `vite.config.ts`
       - In `src/index.css`: `@import "tailwindcss";` and `@import "@xyflow/react/dist/style.css";`

    4. Install shadcn/ui: `npx shadcn@latest init` with:
       - Style: New York
       - Base color: Zinc
       - CSS variables: yes
       - `components.json` pointing to `src/components/ui`

    5. Add required shadcn components: `npx shadcn@latest add card badge scroll-area separator sheet`

    6. Configure Bloomberg dark theme in `src/index.css`:
       - Set `:root` / `.dark` CSS variables for zinc-based dark palette
       - Add `.bloomberg-flow` CSS class overriding React Flow CSS variables:
         ```css
         .bloomberg-flow {
           --xy-background-color-default: #0a0a0b;
           --xy-node-background-color-default: #18181b;
           --xy-node-border-default: 1px solid #3f3f46;
           --xy-edge-stroke-default: #3f3f46;
           --xy-edge-stroke-selected-default: #0A84FF;
           --xy-handle-background-color-default: #0A84FF;
         }
         ```
       - Set `body` background to `#0a0a0b`, font-family to Inter (import from Google Fonts in index.html)

    7. Set up `src/lib/utils.ts` with `cn()` helper (clsx + tailwind-merge) — standard shadcn pattern.

    8. Configure `vite.config.ts` with proxy for API requests:
       ```ts
       server: {
         proxy: {
           '/api': 'http://localhost:8000'
         }
       }
       ```

    Ensure `npm run dev` starts successfully and renders a blank dark page.

  <name>Task 2: Zustand store, SSE hook, dagre layout, React Flow pipeline graph, three-panel layout</name>
  <files>
    frontend/src/lib/dagre.ts

  <action>
    1. **Dagre layout utility** (`src/lib/dagre.ts`):
       - `getLayoutedElements(nodes, edges, direction='LR')` function
       - NODE_WIDTH=172, NODE_HEIGHT=56
       - Uses `@dagrejs/dagre` with rankdir, ranksep=80, nodesep=40
       - Returns nodes with computed positions and sourcePosition=Right, targetPosition=Left

    2. **Custom node components** (define outside render, memo-wrapped):
       - `AgentNode` (`src/components/graph/nodes/AgentNode.tsx`): Renders agent persona name, status badge (idle/running/complete/error), last result preview. Uses Handle left (target) and right (source). Status-driven border colors: idle=zinc-700, running=blue-500 + animate-pulse, complete=emerald-500, error=red-500. Background: zinc-900 base.
       - `GateNode` (`src/components/graph/nodes/GateNode.tsx`): Similar to AgentNode but with a shield/gate icon, label "Quality Gate", hexagonal or diamond-shaped border accent to visually distinguish from agent nodes.
       - `StageNode` (`src/components/graph/nodes/StageNode.tsx`): For scanner, signal detector, committee, CIO nodes. Simpler card with label and status dot.

    3. **Zustand store** (`src/store/pipelineStore.ts`):
       - Define initial pipeline nodes (10 nodes total):
         - `scanner` (StageNode) — "Market Scanner"
         - `signal_detector` (StageNode) — "Signal Detector"
         - `quality_gate` (GateNode) — "Quality Gate"
         - `buffett` (AgentNode) — "Warren Buffett"
         - `munger` (AgentNode) — "Charlie Munger"
         - `ackman` (AgentNode) — "Bill Ackman"
         - `cohen` (AgentNode) — "Steve Cohen"
         - `dalio` (AgentNode) — "Ray Dalio"
         - `committee` (StageNode) — "Committee"
         - `cio` (StageNode) — "CIO"
       - Define initial edges: scanner->signal_detector->quality_gate->[5 agents]->committee->cio (quality_gate fans out to all 5, all 5 fan in to committee)
       - Run `getLayoutedElements()` once to compute positions
       - Store state: `nodes`, `edges`, `feedItems: FeedItem[]`, `outputRankings: Opportunity[]`
       - Store actions: `onNodesChange`, `updateNodeStatus(nodeId, status, data?)`, `addFeedItem(item)`, `setOutputRankings(rankings)`, `handleSSEEvent(eventType, data)`
       - `handleSSEEvent` dispatches based on event type:
         - `AGENT_STARTED` -> updateNodeStatus(agent_id, 'running')
         - `AGENT_COMPLETE` -> updateNodeStatus(agent_id, 'complete', result)
         - `COMMITTEE_COMPLETE` -> updateNodeStatus('committee', 'complete')
         - `DECISION_MADE` -> updateNodeStatus('cio', 'complete'), add to feedItems and outputRankings
       - Export `usePipelineStore`

    4. **SSE hook** (`src/hooks/usePipelineSSE.ts`):
       - `usePipelineSSE(url: string)` — creates EventSource in useEffect
       - Listens for `pipeline` event (matching backend SSE event name from events.py)
       - Parses `e.data` as JSON, extracts `event` and `data` fields
       - Calls `store.handleSSEEvent(event, data)`
       - `es.onerror`: only log warning, let browser auto-reconnect
       - Cleanup: `es.close()` on unmount
       - Note: The backend sends all events as `event: pipeline` with the event_type inside the JSON data blob. Parse accordingly.

    5. **Three-panel layout** (`src/components/layout/DashboardLayout.tsx`):
       - CSS Grid: `grid-template-columns: 1fr 320px 380px` with `height: 100dvh`
       - Left panel: Pipeline graph (takes remaining space)
       - Middle panel: Opportunity feed (320px)
       - Right panel: Output dashboard (380px)
       - All panels: `overflow-y: auto`, dark borders between them (`border-zinc-800`)
       - Top header bar: "HedgeFund V2 — Alpha Discovery Engine" in font-mono, small, zinc-400 text. Optional green dot if SSE connected.

    6. **Pipeline graph** (`src/components/graph/PipelineGraph.tsx`):
       - Import `ReactFlow, Background, Controls` from `@xyflow/react`
       - Subscribe to `nodes` and `edges` from store via selectors
       - Define `nodeTypes = { agent: AgentNode, gate: GateNode, stage: StageNode }` at module level (NOT inside render — this is the #1 React Flow pitfall)
       - `edgeTypes` can be default for now (04-02 adds animated edges)
       - Pass `colorMode="dark"`, `fitView`, `fitViewOptions={{ padding: 0.2 }}`, `className="bloomberg-flow"`
       - Include `<Background color="#27272a" gap={24} />` and `<Controls />`

    7. **Placeholder panels**:
       - `OpportunityFeed.tsx`: Renders "Live Feed" header + "Waiting for events..." placeholder in ScrollArea. Will be fully built in 04-03.
       - `OutputDashboard.tsx`: Renders "Top Opportunities" header + "No opportunities yet" placeholder in ScrollArea. Will be fully built in 04-04.

    8. **Wire App.tsx**: Import DashboardLayout, call `usePipelineSSE('/api/v1/events/stream')` at top level, render `<DashboardLayout />`.

    IMPORTANT anti-patterns to avoid:
    - Do NOT define nodeTypes/edgeTypes inside a component render function
    - Do NOT create multiple EventSource instances
    - Do NOT re-run dagre on every render — compute once at store init
    - Do NOT subscribe panel components to the full `nodes` array — use slice selectors
  </action>
