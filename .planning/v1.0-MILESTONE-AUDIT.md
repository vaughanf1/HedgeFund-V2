## Critical Gaps

### 1. `consume_queue` never started — Phase 2→3 bridge broken

**Files:** `backend/app/tasks/analyse_opportunity.py`, `docker-compose.yml`, `backend/app/tasks/celery_app.py`

`consume_queue` is the BLPOP consumer that bridges the Redis opportunity queue (Phase 2) to agent fan-out (Phase 3). It is defined as a Celery task but:
- Not in `beat_schedule`
- No startup command in `docker-compose.yml` invokes it
- `celery_app.py` has no `autodiscover_tasks()` or `include` list — task modules may not be registered

**Impact:** Flow 1 completely broken. Signals queue up in Redis but nothing pops them.

### 2. `DECISION_MADE` SSE payload mismatch — frontend reads fields backend never sends

**Files:** `backend/app/tasks/analyse_opportunity.py` (lines 349-360), `frontend/src/store/pipelineStore.ts` (lines 172-234)

Backend sends flat keys: `{opportunity_id, final_verdict, conviction_score, ...}`
Frontend expects nested: `data['decision']['final_verdict']`, `data['ticker']`, `data['verdicts']`

**Impact:** SSE live-update path is silently dead. No feed items created, no output rankings updated via SSE.

### 3. `AGENT_STARTED` SSE field name mismatch

**Files:** `backend/app/tasks/analyse_opportunity.py` (lines 118-121), `frontend/src/store/pipelineStore.ts` (lines 141-144)

Backend sends `persona: "buffett"`. Frontend reads `agent_id`. Agent nodes never transition to `running` state.

### 4. Information asymmetry bypassed — DataPartitioner receives wrong data shape

**Files:** `backend/app/agents/partitioner.py`, `backend/app/tasks/analyse_opportunity.py`

`DataPartitioner` expects keys like `fundamentals`, `price_action`, `news`, `insider_trades`. The actual data passed is the raw opportunity dict with `signals` list. Partitioner passes unrecognised keys through — every persona gets identical data.

**Impact:** AGNT-07 (information asymmetry) functionally unsatisfied despite code existing.

### 5. `CIODecision` schema missing fields frontend expects

**Files:** `backend/app/agents/schemas.py`, `frontend/src/hooks/useOpportunities.ts`, `frontend/src/components/inspect/OpportunitySheet.tsx`

Missing from schema: `ticker`, `summary`/`cio_summary`, `expected_upside`. Frontend shows compound ID (`AAPL:2026-03-26T06:00:00`) instead of ticker symbol. CIO summary always empty.

### 6. Feed item click → Sheet always 404

**Files:** `frontend/src/components/feed/FeedItem.tsx`, `frontend/src/store/pipelineStore.ts`

Feed items use synthetic IDs (random UUID or `agentId-ticker-timestamp`), not the compound `ticker:detected_at` key the backend expects. Sheet fetch always returns 404.

## Moderate Issues

- **Docker Compose missing frontend service** — `docker compose up` doesn't start Vite; must run `npm run dev` manually
- **`sector_momentum` silently disabled** when `SECTOR_MAP` env var not set (no warning log)
- **Dashboard SSE indicator hardcoded to DISCONNECTED** — never shows connected state
- **Verdict enum mismatch** — backend uses `INVEST`/`PASS`, frontend handles `BUY`/`SELL` — falls to unstyled default

## Tech Debt

- FMP endpoint paths low-confidence (D-01-02-3) — must validate before real API calls
- COST_PER_MTOK hardcoded — needs update mechanism when Anthropic changes pricing
- session.merge() not load-tested on large OHLCV batches
- Signal scoring thresholds empirical — need false-positive instrumentation
- `/api/v1/signals` endpoints orphaned (no frontend consumer)

## Recommendations

The 6 critical gaps prevent the system from functioning end-to-end at runtime. Code exists for all 47 requirements but the cross-phase wiring has integration defects that make the pipeline non-functional without fixes.
